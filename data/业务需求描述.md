背景与痛点
业务场景描述
面向 教育与学习场景，旨在构建一个以知识点为核心、支持多教材并存、能够进行前置关系推理与学习路径规划的知识图谱系统。
在真实教学与学习过程中，同一学科通常存在多种教材、课程与教学资源，不同教材在章节划分、知识点顺序、讲解深度上存在显著差异。然而，学习者真正需要掌握的是稳定的知识点体系，而不是某一本教材的章节结构。因此，本系统的核心业务目标是：
将教材结构与知识体系解耦，构建统一的知识本体，并在此基础上支撑智能学习与教学应用。
1. 多教材并存的教学现实
在同一学科下：
●同一个知识点：
●在教材 A 中可能是「核心内容」
●在教材 B 中可能是「选学内容」
●在教材 C 中甚至未出现
●不同教材对知识点的顺序安排不同，但这并不代表认知上的前置关系差异
因此，系统必须满足以下业务要求：
●知识点是全局唯一的语义实体
●教材仅作为知识点的一种教学视图，而非知识本体
●教材顺序不等同于知识点的前置关系（前置关系表达的是认知依赖，而不是教学策略）
2. 以知识点为中心的学习支持
系统中的核心对象是知识点（Concept），每个知识点满足：
知识点是知识体系中的最小单元，通常以命题的形式呈现。命题既可以描述简单的事实，也可以表达复杂的规则、原理或公式 [1-3]。作为构成知识的基础，知识点通过命题学习的方式建立多个概念之间的联系
在此基础上，系统需要支持：
●知识点层级结构
●上层知识点表示更抽象的概念
●下层知识点表示更具体的方法或技能
●知识点前置关系
●该知识点的资源【PPT、讲义、code、视频（不同类型）】
●明确学习某一知识点前必须掌握哪些同层级知识点
●用于学习路径规划与薄弱点诊断
3. 面向智能应用的业务能力
在构建统一知识图谱后，系统将支撑多种智能教育应用，包括但不限于：
✅ 自动生成学习路径
将教材章节或课程内容映射为知识点路径，生成符合认知逻辑的学习顺序，而不是简单复用教材目录结构
✅ 跨教材补救学习
当学习者在某一本教材中遇到理解困难时，系统可以在其他教材中定位相同知识点的替代讲解或补充资源，实现“换教材不丢进度” 
✅ 精准定位学习薄弱点
通过错题、练习或评测结果，定位对应知识点，并进一步追溯其前置知识点，判断学习断裂发生的位置
4. 支持持续演进的知识体系
教学资源与教材内容会持续更新，因此系统需要具备：
●增量更新能力新增文本或资源时，只处理新增内容，不重复消耗已有计算结果
●可演进的知识图谱结构在不破坏既有知识结构的前提下，逐步补充新的知识点与关系
为此，系统采用“事件驱动 + 全局裁决”的方式，将文本作为证据输入，而将知识点结构作为稳定输出，从而保证长期一致性与可维护性。

实体
KnowledgePoint（知识点）
- id: string              # 全局唯一（推荐可读 id）
- name: string            # 知识点名称
- level: string           # L1 | L2 | L3（核心 / 子概念 / 技能）
- description: string
- difficulty: float       # 0~1
- aliases: [string]
Resource（资源）
"""资源，知识图谱中的资源节点基类。"""
id: str = Field(..., description="资源唯一标识符")
url: str = Field(..., description="资源URL地址")
resource_type: str = Field(..., description="资源类型")
