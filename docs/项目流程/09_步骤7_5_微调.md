# 步骤7.5：微调（`tune_graph`）

对应实现：`knowledge_graph/agents/refinement.py`  
提示词：`prompts/Refinement_Prompt.txt`

## 架构流程图

```mermaid
flowchart TD
    A[读取 evaluation.adjustment_suggestions] --> B[输入裁剪(高优先级+TopN)]
    B --> C[生成微调计划(max_actions 上限可配置)]
    C --> D[动作排序: MERGE→ADD→DELETE]
    D --> E[动作解析: 解析 keep/remove/parent/target id]
    E --> F[执行动作]
    G --> H[结构检查(新增节点是否孤立)]
    H --> I{健康?}
    I -- 否 --> J[回滚当前动作]
    I -- 是 --> K[保留变更并更新state]
```

## 详细实现说明

- **输入**
  - `state.evaluation_report.adjustment_suggestions`
  - 当前最新 `state.calibrated_*` 图数据。
- **计划生成阶段**
  - 只保留高优先级建议并做数量裁剪，降低 LLM 阻塞风险。
  - 输出动作上限控制在 `refinement.max_actions`（代码默认 35，可配置）。
- **执行阶段硬约束**
  - 当前实现支持三类动作：`MERGE` / `ADD` / `DELETE`（并非仅 ADD）。
  - 动作执行顺序固定为：**MERGE → ADD → DELETE**（见 `_ACTION_ORDER`）。
  - ADD 的父节点可以是：
    - `L1`：补充 `L2`（受 `L1` 下 `L2<=10` 约束）
    - `L2`：补充 `L3`（受 `refinement.max_l3_per_l2`、`refinement.max_l3_add_per_call` 等约束）
- **一致性与回滚**
  - 先做关系合法性预清洗，建立稳定基线。
  - 每个动作后检查结构指标，若新增节点产生孤立则回滚动作。
- **输出**
  - 更新 `state.calibrated_kps`、`state.calibrated_relationships`
  - 写回 `data/output/calibrated_entities.parquet` 与 `calibrated_relationships.parquet`
  - `state.tuning_summary` 记录动作执行结果。
