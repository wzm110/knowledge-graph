# 文档与代码对照表（以代码为准）

本表用于将 `docs/**` 与仓库根部说明文档的描述，**逐条对齐到 `knowledge_graph/**` 的真实实现**。  
当文档与代码不一致时，优先更新文档；如遇“历史规划/已废弃实现”，需在文档中明确标注并指向当前入口。

## 1. 总入口与流程控制

- **主入口（CLI）**：`knowledge_graph/__main__.py`
  - **step 位置参数**：`full|extract_l1|validate_l1|extract_l1_rels|extract|vectorize|calibrate|evaluate|build`
  - **关键开关**：
    - `--max-loops`：L1 提取↔验证循环上限
    - `--max-eval-loops`：评测↔微调循环上限
    - `--incremental`：增量处理（影响 L1、步骤3等是否跳过/回读）
    - `--full-refresh-l1`：强制重跑步骤 1-2（否则可能跳过并复用 `stage1_entities.parquet`）
- **主流程（真实执行顺序/循环与落盘）**：`knowledge_graph/pipeline.py` → `run_full_pipeline()`
  - 主干顺序：`1-2(L1循环)` → `3` → `4` → `5` → `6` → `6.5` → `7↔7.5(循环)` → `8`

## 2. 分步骤文档对应的代码锚点

### Step 1：L1 提取

- **文档**：`docs/项目流程/01_步骤1_L1提取.md`
- **代码**：`knowledge_graph/agents/l1_extractor.py`
- **落盘**：`data/output/stage1_entities.parquet`（初版/后续会被步骤2覆盖为验证版）
- **关联开关**：`--max-loops`（与步骤2一起形成循环）、`--full-refresh-l1`

### Step 2：L1 验证（可回环）

- **文档**：`docs/项目流程/02_步骤2_L1验证.md`
- **代码**：`knowledge_graph/agents/l1_validator.py` + `knowledge_graph/pipeline.py`（循环控制在 pipeline）
- **关键状态字段**：`validation_errors`、`validated_l1_concepts`
- **落盘**：`data/output/stage1_entities.parquet`（验证版覆盖）

### Step 3：L1 前置关系提取

- **文档**：`docs/项目流程/03_步骤3_L1前置关系提取.md`
- **代码**：`knowledge_graph/agents/l1_prerequisite.py`
- **落盘**：`data/output/stage2_relationships.parquet`

### Step 4：实体与关系抽取（含增量/回读逻辑）

- **文档**：`docs/项目流程/04_步骤4_实体与关系抽取.md`
- **代码**：`knowledge_graph/agents/entity_extractor.py`
- **输入**：`data/input/*.csv`（排除 `*目录.csv`），以及 `data/processed_files.json`
- **落盘**：
  - `data/output/stage3_entities.parquet`
  - `data/output/stage3_relationships.parquet`
  - `data/output/stage3_resources.parquet`
- **关键语义**：
  - 无新增章节时：直接回读 stage3 三份 parquet，跳过 LLM
  - 资源仍会落盘，但后续阶段可能不入图/不进 Neo4j（见 Step 6/7/8）

### Step 5：向量化（当前仅知识点，不含资源）

- **文档**：`docs/项目流程/05_步骤5_向量化.md`
- **代码**：`knowledge_graph/agents/vectorization.py` + `knowledge_graph/utils/vector_db.py`
- **关键语义**：只向量化 `knowledge_points`（资源忽略）

### Step 6：数据校准（关系合法性 + 去重 + 资源处理）

- **文档**：`docs/项目流程/06_步骤6_数据校准.md`
- **代码**：`knowledge_graph/agents/calibration.py`
- **关键语义**：
  - `validate_and_calibrate_relationships()` 约束 `contains` 只能是 `L1->L2|L2->L3|L3->L4`
  - 校准产物写入 `calibrated_entities/relationships.parquet`
  - **校准阶段资源不入图**，`calibrated_resources` 通常为空（资源仍保留在 stage3_resources）
- **落盘**：
  - `data/output/calibrated_entities.parquet`
  - `data/output/calibrated_relationships.parquet`

### Step 6.5：L2 重聚合（下沉 + 新建聚合 L2 + id 前缀规范化）

- **文档**：`docs/项目流程/07_步骤6_5_L2重聚合.md`
- **代码**：`knowledge_graph/agents/recluster.py`
- **关键语义**：
  - old L2 → L3、old L3 → L4
  - 关系重写：删除 `L1->oldL2 contains`，新增 `L1->newL2 contains`、`newL2->oldL2 contains`
  - 下沉后会执行 `_apply_sink_id_renames()`：统一将下沉节点 id 前缀规范为 `l3_` / `l4_`
- **落盘**：覆盖更新
  - `data/output/calibrated_entities.parquet`
  - `data/output/calibrated_relationships.parquet`

### Step 7：评测（含按簇评测与最终评分）

- **文档**：`docs/项目流程/08_步骤7_评测.md`
- **代码**：`knowledge_graph/agents/evaluation.py`
- **关键语义**：
  - 评测输入会过滤 `has_resource`（资源不参与评测与入库）
  - 输出包含全局评测 + 各 L1 簇评测，最终计算 `final_score`
  - 通过判定：基于 `final_score`、`blocking_issues`、最低簇分等综合逻辑（以代码为准）
- **落盘**：
  - 历史目录：`data/output/evaluation/<run_id>/*.json|*.md`
  - 最新快照：`data/output/evaluation/latest.json`
  - 最终评估同步：`data/output/final_evaluation/**`（含 `clusters/*.md`）

### Step 7.5：微调（支持 ADD/DELETE/MERGE）

- **文档**：`docs/项目流程/09_步骤7_5_微调.md`、`docs/图谱微调操作规范.md`
- **代码**：`knowledge_graph/agents/refinement.py`
- **关键语义**：
  - 动作顺序：`MERGE → ADD → DELETE`
  - ADD 既可补 `L1->L2`，也可补 `L2->L3`（受上限约束：L1 下 L2 ≤10；L2 下 L3 有单独上限配置）
  - 每个动作后会校验关系并可能回滚（以代码中“孤立节点回滚”等规则为准）
- **落盘**：覆盖更新
  - `data/output/calibrated_entities.parquet`
  - `data/output/calibrated_relationships.parquet`

### Step 8：Neo4j 构建（当前仅知识点与非资源关系）

- **文档**：`docs/项目流程/10_步骤8_Neo4j构建.md`
- **代码**：`knowledge_graph/agents/graph_builder.py`
- **关键语义**：
  - 入库前会过滤 `has_resource`，仅写入知识点与知识点间关系
  - 会清空库：`MATCH (n) DETACH DELETE n`
  - 支持写入 L1/L2/L3/L4 节点
